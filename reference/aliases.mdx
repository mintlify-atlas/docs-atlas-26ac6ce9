## Overview

Shell aliases are defined across multiple files in `system/zsh/`:
- `aliases.zsh` - General aliases and utilities
- `git.zsh` - Git-specific aliases and functions
- `pacman.zsh` - Package manager aliases
- `functions.zsh` - Custom shell functions

### Package Functions

<Accordion title="upgrade - Safe system upgrade">
```bash
function upgrade() {
  sudo pacman -Sy
  echo ":: Checking Arch Linux PGP Keyring..."
  local installedver="$(LANG= sudo pacman -Qi archlinux-keyring | grep -Po '(?<=Version         : ).*')"
  local currentver="$(LANG= sudo pacman -Si archlinux-keyring | grep -Po '(?<=Version         : ).*')"
  if [ $installedver != $currentver ]; then
    echo " Arch Linux PGP Keyring is out of date."
    echo " Updating before full system upgrade."
    sudo pacman -S --needed --noconfirm archlinux-keyring
  else
    echo " Arch Linux PGP Keyring is up to date."
    echo " Proceeding with full system upgrade."
  fi
  paru -Su
}
```

Safely upgrades the system by checking and updating the PGP keyring first to prevent signature errors.
</Accordion>

### Git Helper Functions

<AccordionGroup>
  <Accordion title="git_current_branch - Get current branch name">
    ```bash
    function git_current_branch() {
      git symbolic-ref --short HEAD 2>/dev/null || \
      git rev-parse --short HEAD 2>/dev/null || return 1
    }
    ```
    Returns the current branch name or short commit hash.
  </Accordion>

  <Accordion title="git_main_branch - Detect main/master branch">
    ```bash
    function git_main_branch() {
      git rev-parse --git-dir &>/dev/null || return
      
      local ref
      for ref in refs/{heads,remotes/{origin,upstream}}/{main,trunk,master}; do
        if git show-ref -q --verify $ref 2>/dev/null; then
          echo ${ref:t}
          return 0
        fi
      done
      
      echo master
      return 1
    }
    ```
    Intelligently detects whether the repo uses `main`, `master`, or `trunk`.
  </Accordion>

  <Accordion title="git_develop_branch - Detect develop branch">
    ```bash
    function git_develop_branch() {
      git rev-parse --git-dir &>/dev/null || return
      
      local branch
      for branch in dev devel develop development; do
        if git show-ref -q --verify refs/heads/$branch 2>/dev/null; then
          echo $branch
          return 0
        fi
      done
      
      echo develop
      return 1
    }
    ```
    Detects the development branch name.
  </Accordion>
</AccordionGroup>

## Custom Shell Functions

<AccordionGroup>
  <Accordion title="mkcd - Create directory and cd into it">
    ```bash
    mkcd() {
        mkdir -p "$1" && cd "$1"
    }
    ```
    Creates a directory (including parents) and changes into it.
    
    **Usage:** `mkcd ~/projects/new-project`
  </Accordion>

  <Accordion title="bak - Quick backup of a file">
    ```bash
    bak() {
        cp "$1" "$1.bak"
    }
    ```
    Creates a `.bak` copy of a file.
    
    **Usage:** `bak important-file.txt`
  </Accordion>

  <Accordion title="up - Navigate up multiple directory levels">
    ```bash
    up() {
        local d=""
        local limit=${1:-1}
        for ((i=1 ; i <= limit ; i++)); do
            d="${d}../"
        done
        cd "$d"
    }
    ```
    Go up N directory levels.
    
    **Usage:** `up 3` (goes up 3 levels)
  </Accordion>

  <Accordion title="down - Navigate down using fzf">
    ```bash
    down() {
        if ! command -v fd &> /dev/null || ! command -v fzf &> /dev/null; then
            echo "down: requires 'fd' and 'fzf'" >&2
            return 1
        fi

        local depth=${1:-1}
        local target=$(fd --type d --max-depth "$depth" . | fzf)
        [[ -n "$target" ]] && cd "$target"
    }
    ```
    Fuzzy find and navigate to a subdirectory.
    
    **Usage:** `down 3` (search 3 levels deep)
  </Accordion>
</AccordionGroup>
